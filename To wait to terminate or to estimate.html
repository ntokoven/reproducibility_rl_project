üßê N-step Actor-Critic: the definitive Policy Gradient?

Policy gradient is a widely used approach to Reinforcement Learning problems, in part, given how conveniently they be integrated within deep learning frameworks that have dominated the AI efforts in the past decade. More precisely, actor-critic methods have become a highly popular choice given how these improve vanilla implementations of policy gradient, which typically present serious drawbacks such as high variance and slow convergence.

Our motivation for investigating them in comparison to Monte Carlo methods (e.g. REINFORCE) is reasoned with the assumption that reducing variance might be very helpful for environments, where agent struggles to terminate following initial bad policy.

In this post, we'll address the problem of learning either from sampled episodes or in online manner, as well as, dive into the particular case of n-step bootstrapping actor-critic and analyse the benefits and disadvantages it brings to the table.

üë∂ The basics: Policy Gradient

Policy gradient methods are the powerful tool to use for directly learning the optimal policy when the environment is unknown. Especially it works well for cases with huge number of possible states. For such problems we should approximate the distribution of actions by parametrizing it with the set of trainable parameters. The main advantage that policy gradient introduces is not needing to consult a value function in order to determine an agent's actions, that is useful in many scenarios where the value function is not discrete or hard to estimate. By parametrizing the policy as a continuous function and defining a performance measure, we can use gradient methods to optimise it, which proves a great match with Neural Nets and the well known Deep Learning technique of backpropagation.

‚ñ∂Ô∏è Why Actor-Critic?

In short, Actor-Critic leverages the bias introduced by bootstrapping to reduce the variance in training. In other words, they actively use online the current policy and an estimation of the value function to update the policy and value function parametrizations, instead of having to run a full episode each time. Although Monte Carlo approaches such as REINFORCE are guaranteed to converge to a local maximum given enough samples and have zero bias, they tend to do so very slowly, so trading bias for variance and speed of convergence seems like a valid goal to pursue.

‚è≠ What about n-step bootstrapping?

As we just saw, One-step Actor-Critic uses one step ahead, like Temporal Difference (TD) algorithms, to estimate the returns, and Monte Carlo uses a whole episode, which begs the question: what about an in-between? Alternatively, to what degree are we willing to push this bias/variance trade-off?

N-step Actor Critic answers this question: instead of using only one step ahead, we can use n-steps ahead. By doing this, we can explore different levels of tradeoff which might be more more or less appropriate for certain environments.

‚úåÔ∏è Generalized Advantage Estimation: a technique to succeed

The Generalized Advantage Estimation or GAE is a novel policy gradient method that combines both TD and Monte Carlo approach. üß™ Experimental Design

Unfortunately, reproducing results for state-of-the-art deep RL methods is seldom straightforward. In particular, non-determinism in standard benchmark environments, combined with variance intrinsic to the methods, can make reported results tough to interpret. (Deep RL that Matters)

In order to compare multi/1-step GAE with Reinforce we had to define experiment conditions being same for every candidate. In all cases the choice of Policy Network has fallen to be a 2-layer MLP (Dense+ReLU‚ÜíDense+SoftMax). Adam optimizer with learning rate of 0.01 was used. All agents were trained as to be discounted with Œ≥=0.99 in order to give higher importance to the current state but still make it in a way to remain being strongly influenced by estimated future returns. We build our experiments in continuous state-space environments : CartPole (source) and Acrobot (source) - in order to focus on the effects arising in an environment where high exploration is not required. Results presented in this blogposts are obtained from running 10 agents with 3 different seeds each for 500 training episodes.

üìà How to tell which one is better?

The first idea that pops into everyone's mind is to see the maximum value that each trained algorithm can reach. Is it enough to tell which performs the better? We don't think so, thus we decided to dig into the mysterious world of comparing RL algorithms. Well, we found 4 great metrics in a recent master thesis, so we just adapted them, here is our version:

  • Highest and mean score: most intuitive ways of evaluation
  • Time complexity: for a fixed number of steps the average time needed for training
  • Sample efficiency: the average time that the algorithm took to reach the maximum score
  • Robustness: the standard deviation of scores for trained agents

As we can see, in all cases multi-step GAE outperformed its Monte Carlo opponent REINFORCE, as well as, its 1-step analogue. For CartPole it can be clearly seen how less amount of variance is produced by the algorithm. In all cases agents were able to learn within reasonable time, so we can suppose that differences in run-time are occurred due to the implementation properties but not the crucial differences in algorithms structure. We can see that on average GAE appears to be more data-efficient, because of the steeper increase in performance over first episodes. For us it means that we can reach somehow more reliable agents when being constrained on the data amount/computational power.

Conclusions

In this blog-post we explained how to build agents in environments with continuous state space using Policy Gradient methods, as well as, provided our assumptions with corresponding experiments. We also formulated problem-specific metrics to compare the competing algorithms performance. After all, from our experiments we can state that training agents using bootstrapping improves agents average returns. Also we can conclude that using 1-step bootstrapping is a strict way to reduce the variance. Multi-step agents showed to perform better, since introducing reasonable amount of bias in solutions of RL problems might increase their quality, especially in environments where the terminal state is hard to be reached within reasonable time when following random policy.
